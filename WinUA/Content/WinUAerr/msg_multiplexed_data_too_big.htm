Data Will Not Fit in Buffer

The multiplexed data from this file will not fit in the data buffer. Some data points will be dropped. To prevent dropping of data, close all open data files and increase the data buffer size to a minimum of xxx points.

 

Problem:

The multiplexed data file contains too many points and must be compressed to fit into the current data buffer. Compressing the multiplexed data may give undesirable results because excess data points are discarded prior to being sorted by the multiplexed value (e.g., frequency). In some cases the compression process will produce unequally spaced data points resulting in a misshapen curve.

 

Solution:

Close all open data files. Select File/Main Options, then click the Data page. Enter at least the number of minimum points given in the message as the Data Buffer Size. A larger number is acceptable. Click the OK button. Then reopen the desired multiplexed data file. The entire file should be plotted as desired.

 

If this error occurs frequently, the Data Buffer Size should be permanently set to a larger number by pressing the Save button on the Main Options  Data Page after entering a new buffer size.
