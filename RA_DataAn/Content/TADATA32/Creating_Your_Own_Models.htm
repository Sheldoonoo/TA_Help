Creating Your Own Models

 

In addition to the pre-defined models available to you, the data analysis package allows you to create your own model definitions. These can be exported (saved to file) and imported (opened) by other users.

The model fit editor creates the function using Microsoft® Visual Basic® Scripting Edition (VBScript) and so you can use any of the functions contained in the VBScript package.

Procedure to Create a Model

To create a new column, first load the required data file and switch to graph view.

  1. Select Analysis on the menu.
  2. Click on User Defined Models.
  3. Click on Define.
  4. The new model wizard is started.
  5. Enter a Name for this new column variable.
  6. Select the type of data the model will apply to. You can select the Any option for a generic model, but you must then ensure that it only contains variables that occur in all data types.
  7. Click the Next button
  8. Create the required model. Add existing variables to the function by selecting them in the right hand box and either double click or use the << button.

NOTE: You can type the variables directly, but it is generally better to select from the list to ensure the correct spelling.

  1. Any constants you wish to use must be named C1, C2 etc. You will be prompted to define these later in the wizard
  2. The function can contain any numerical operators ( + - * / ) as well as any of the mathematical functions available to Microsoft® Visual Basic® Scripting Edition (VBScript).
  3. As in the standard models, it is usual to define what must be on the X and Y axes of the graph in order for the model fit to be displayed. Set these using the Y Variable and X Variable entries. You can then use the terms X and Y in your model definition.
  4. Click the Next button
  5. Enter in descriptive information for the variables you have used (C1, C2 etc). You will need to click the Next button to define each one in turn.

Once you have defined the model, it can be accessed from the standard model lists.
