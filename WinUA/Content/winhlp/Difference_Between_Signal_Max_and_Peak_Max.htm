Difference Between Signal Max and Peak Max

Signal Max and Peak Max (Peak Integration Height) appear to report the same analysis result, but they are actually different analyses. The result provided by each is highly dependant on the baseline selected.

 

Analyzing for Signal Max reports the largest excursion of the data from a horizontal baseline placed halfway between the two cursor positions specified in the analysis. Analyzing for Peak Max reports the largest excursion of the data from a sloping baseline drawn directly between the two cursor positions as they intersect the data. When the baseline is horizontal, both analyses give the same result. But, when the Peak Max baseline is sloping it can give a different result than Signal Max. Typically, the more the Peak Max baseline slopes, the greater the difference in results.

 

Analyzing for Peak Max gives the correct result for baseline resolved peaks. Signal Max does not (unless the baseline happens to be horizontal). Signal Max analysis is really intended for determining the largest value of a signal over a range, not the signal's maximum distance from a baseline. Signal Max is typically used in DMA analysis. Peak Max is typically used in DSC and DTA analysis.

 

When the Signal Max analysis is applied to a step transition, Universal Analysis can become confused about which way to search for the maximum value because the program is looking both above and below a horizontal baseline drawn at the mid-point between the cursors. When the cursors are placed before and after a step transition, the signal maximum can be found at a point that is actually the numeric signal minimum, because this point is further away, in absolute value terms, from the horizontal baseline than the data point with the largest numeric value. In this case the Signal Max function is providing a "Signal Min" result (e.g., for negative going data). This result typically only occurs with step transitions and linear signal ramps.
