Using the Database Search Wizard

 

If you are having trouble locating the record(s) desired, you can access the Database Search Wizard. The wizard uses SQL (Structured Query Language) to search the relational database.

Select the Database Search Wizard button,   , on the Database Viewer window. The Find Records in Database window (shown below) is displayed.

This topic will provide a brief introduction to the use of this search function.  For more details consult references on SQL.

Locating Records Using SQL

There are many different ways to search the database for records using Database Search Wizard, basically you will need to set up search criteria that allow you located data based on specific parameters that you request.  For example, you can request data that was created between two different dates.  Searching for data using SQL is very much like defining a mathematical equation.  

Each field provides a drop-down list that you can choose from.

  • Field Name: Lists the identification field that will be used in the search.
  • NOT: A check box to either search to include the criteria (e.g., search for data equal [=] to parameters) or to exclude the criteria (e.g., search for data NOT equal [=] to parameters).
  • Criteria: Lists the search conditions, also called query predicates that restrict the data that will be returned.  Here are some operators used in the Criteria:

Operator

Comparison

=  

Specifies that the fields are of equal value.

LIKE

Specifies that the fields match a specified string pattern. The wizard will automatically include wild card letters "%" surround the string you entered in the Example field. Case is significant.

>=

Specifies that the field must be greater than or equal to the value in the example text.

<=  

Specifies that the field must be less than or equal to the value in the example text.

Between

Specifies that the field must be greater than or equal to the lower value and less than or equal to the higher value in the example text. For a text filed, when you select Between-operator, a string "?? AND ??" will appear in Example field. Just replace the ?? with two values. For a Date field, Click on the Example field will display a dialog boxes to let you enter two date values.

By combining the NOT check box, you can also get addition comparisons:

 

Operator

Comparison

NOT <=

Specifies that the field must be greater than the value in the example text.

NOT >=

Specifies that the field must be less the value in the example text.

NOT =

Specifies that the field must not equal the value in the example text.
  • Example:  Enter an example of the desired search parameter.  For example, if you want to search by instrument you might select "Module" in Field Name combo-box, select "Like" in Criteria combo-box then enter "DSC" in this field.
  • Case:  A check box to chose whether your parameters will be case-sensitive (checked) or not (blank).
  • Logical:  Lists <None>, AND, and OR.  <None> is used for a single search condition. AND and OR are used to combine multiple search filters. For example, you can combine the following SQL filters by using an AND:

mInstserial = 0100-RPP0 AND   

The first search condition

mUSER_NO = 1021493233

The second search condition
  • SQL Filter List:  Lists constructed filters.
  • Ins:  A button that inserts a new search filter in between two existing filters.
  • Add:  A button that appends a new filter at the end of the filter list.
  • Del:  A button that deletes a selected filter in the filter list.
  • Show SQL:  Click this button to display the SQL coding that will be used to search the database.
  • Clear All:  Click this button to clear all filters defined in the filter list and start again.

 NOTE:  The database table only stores the ID numbers of projects, user IDs and names. If you select these fields and enter the name of a project for example, the filter in the filter list will automatically convert it to the project ID number.

After you have set up the SQL search parameters, click the Find button.  Then double-click on the desired data file to open it in the Universal Analysis program.
