Creating a Linear Relaxation Modulus File

 

Used to create the linear relaxation modulus, G(t), from molecular weight distribution (mwd) data. You can either generate this from the Rheology Advantage software, or input it from an external source.

Further information can be obtained from W.Thimm, C.Friedrich, M.Marth and J. Honerkamp, Journal of Rheology 43 (1999) 1663 – 1672.

 

Alpha: it is assumed that the h0 = k Ma, where M is the weight average molecular mass, and k and a are constants obtained from experiment. The coefficient, k, is usually very small, the index a (alpha) is about 3.4. a defined in the polymer library as x.

 

Me: this is the entanglement molecular weight, and is given for many polymers in the Polymer library

 

Plateau modulus, GN: this is the modulus in the rubbery regime, i.e. between the glassy regime and the melt. It is used because the glassy modulus is independent of the polymer molecular weight. GN can be obtained approximately from Me = rRT/GN, where r is the sample density, R is the gas constant = 8.314 J mol-1 K-1 in the S.I. system

 

Coefficient: the coefficient used in the G(t) from mwd calculation is defined by l = K Ma, where l lambda is the dominant relaxation time. For practical purposes l = hJss, where Jss is the steady state compliance. K is also given approximately by K = 1.7 k/GN, where k is defined given above.

 

Beta: Theory suggests this should be either 1 or 2; usually 2 is preferred.

 

Kernel: this is defined by the theory used to obtain G(t) from mwd.

 

Start time lapse is the start time for G(t) (non-zero, otherwise cannot be plotted on log scales). 0.01 s would be a reasonable default in most cases.

 

End time lapse is the finish time for G(t). This could be anything form 10 s upwards. 1000 s might be typical.

 

Number of points: number of data points on the G(t) curve to be calculated, e.g. 10 per decade.

 

Integration steps: number of summations performed in the calculation.

 

Mode: i.e. log or linear

 

Cutoff modulus: the modulus value below which the calculation should not be allowed to fall. The default is 10-6

 

Convergence: this defines the point at which the iteration is to cease, since no further progress is being made.
