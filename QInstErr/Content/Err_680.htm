Message 680

Datafile acceptor is not responding.

Problem:

This error is generated because the Datafile Acceptor (DFA) is not communicating to send the data from the instrument to the controller. The communication between the instrument and the DFA were interrupted. the problem could also be that the data file path name specified on the controller Experiment View is not valid or no longer exists.

Solution:

If communication is re-established during the run, no action is required, however, if the problem persists, DO NOT continue any other runs before transferring the current data or the run data will be lost. Try one or more of the following solutions to establish communications between the instrument and controller again.

  • Check the cable connections.
  • Verify the IP address of the instrument to make sure it was not changed.
  • Reselect the filename using the browse button on Experimental View/Summary page.
  • Verify that the destination drive is not full.

After communication is established. Use the Tools/Data Transfer function to transfer the data from the instrument to the controller.

If you cannot establish communication again, contact TA Instruments for help.
