Commands Overview

This section details all the commands that appear in the statement editor of the Rheology Navigator 2 software. Where necessary, the correct syntax for each command is illustrated by an example.

All of the examples show real numbers with a period (.) as the decimal point and a comma (,) as the separator between parameters. The actual character used will depend upon the Country Settings set in the Control Panel. If other characters have been selected, e.g."," as the decimal point character and";" as the list separator, these must be substituted.

Within Rheology Navigator, certain conventions are followed in the naming of commands - click on the titles below to show more information.

  • Comments

    It is possible to add comments in scripts simply by placing a single quote as the first character in the line,

    e.g.

     "This is a script to run a standard oil."

  • Enter

    With commands preceded by Enter, the variables or parameters will be entered at runtime when a particular step is reached in the script (compare this with the Set command). The operator will be presented with a dialog box in which to enter the data at runtime.

  • Get

    Retrieves a value from the instrument, last generated data or a file.

  • Graph

    Commands functions used for graphing.

  • Load

    When a command is preceded by Load, variables or parameters will be recalled to the script by reading in the appropriate file containing the predefined variables. Such variables need to be set up and stored via Rheology Advantage software.

  • Print

    Commands in the statement preceded by Print use the default print settings. Some commands may require the use of an associated Set command.

  • Prompt

    Commands in the statement editor preceded by Prompt will require some user interaction when this step is reached in the script. These must be acknowledged (press OK) before the next step is processed.

  • Set

    e.g.
      SetSample
      LoadSample
      EnterSample

    With commands preceded by Set, variables or parameters are entered within the script at construction time and then set at runtime.
