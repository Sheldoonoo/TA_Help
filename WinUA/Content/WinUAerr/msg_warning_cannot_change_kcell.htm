Cannot Change Cell Constant

The cell constant (Kcell) parameter has changed. The new value will not be applied to the data. Are you sure you want to change the cell constant?

 

Problem:

The Edit Parameter Block function is designed to permit virtually any change to a data file parameter block with the understanding that this function makes an exact copy of the current data file withoutchanging the data in any way. See also: Edit Parameter Block window

 

Some entries in the parameter block stand by themselves and can be changed to anything you like (e.g., Sample, Operator, Run, etc.) Some entries should not be changed to inappropriate values because these entries tell the program how the data was acquired (e.g., Module, Size, Time, etc.) And some entries, such as the cell constant (Kcell), cannot be changed at all because the change must be reflected in the data, and the data is not changed by the Edit Parameter Block function.

 

Solution:

To change the cell constant (Kcell) value, use the Data File Information dialog. This change is permitted because, when you change Kcell on the Data File Information window, Universal Analysis backs out the original Kcell value from the parameter block and then applies the new Kcell value to every data point in the "primary" module signal.
