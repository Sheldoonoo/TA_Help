Running Universal Analysis from a DOS Command Line

Instructions and tips on running the Universal Analysis program from a DOS prompt command are provided below in a question and answer format.

 

Q. Can I get Universal Analysis to automatically enter the filename of my data file when I start the program?

A. Yes. Simply add the name of your data file, separated by a space or comma, on the DOS command line (MS-DOS Prompt) or on the Windows command line (Run Command) following the program name. You may include up to 10 filenames on the command line and Universal Analysis will open each one in succession when the File Open function is requested.

 

Q. Can I run Universal Analysis from a batch job?

A. Yes. Follow the normal conventions for any Windows program.

 

Q. Can I run a macro automatically when I start Universal Analysis?

A. Yes. Include the name of the macro to run in double quotes as the argument to the "/M" program switch. You can also specify the filename of the macro file to load with the "/F" switch. For example, the command:

 

WinUA MyFile.001 /M"My Macro" /F"WinUA.mac"

 

Will run Universal Analysis, enter file MyFile.001, load macro file WinUA.mac, and execute the macro My Macro from the macro file.

 

Tip. If the command line is too long or you want to store the arguments in a separate file, you can specify an indirect command file that contains the command line string (up to a maximum length of 500 characters). On the command line following the program name, specify the filename and path of the command file, preceded by the "@" character. For example:

 

WinUA @\MyData\WinUAcmd

 

Will run Universal Analysis, open the file \MyData\WinUAcmd and process its contents as if they were entered on the command line for WinUA.
