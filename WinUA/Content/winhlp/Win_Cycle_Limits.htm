Cycle Limits Window

Use this window to limit the data used for analysis to a specific data cycle range. This option is valid if you have any data breaks or markers in your file, these can include the following:

 

 Cycle marks (due to Mark Cycle End segments in a procedure)

 Data breaks due to gas switch or event mark (selected on the Main Options  Data Page)

 Other data breaks.

 

Cycle limits count the number of data breaks or cycle marks found in the data file. For multiplexed (e.g., DMA, DEA, µTA, mass spec, etc.) data, the start and end cycle numbers refer to data breaks or cycle marks supplied by the user, not the cycles associated with frequency, amplitude, m/e number, etc.

 

To limit data by the multiplexed value (e.g., frequency, amplitude, m/e number) you must first read in the data over the range of interest using Data Limits, and then select the multiplexed values you want to work with using either Edit/Frequency List, for example, or Edit/Cycle List. For multiplexed data, the Cycle List will include cycles due to both the multiplexed values and user specified data breaks or cycle marks. In this case, the total number of cycles will equal the number of multiplexed values times the number of user specified data breaks or cycle marks.

 

Follow these instructions to limit the data:

  1. Enter the cycle at which you want the program to begin retrieving data.

2. Enter the cycle at which you want the program to stop retrieving data.

3. Check the Reset time to zero box, if you want to reset the Time signal to zero at the start of the specified data limits region.

4. Click on OK.

 

The program scans the data file between the points you specify to determine the minimum and maximum values, which are then used to determine the autoscaling factors for the plot. Any data outside these limits is not plotted or analyzed, regardless of the scaling factors you select later.

 

To regain the entire data limits for analysis, use the Data Limits/Full Range option.
