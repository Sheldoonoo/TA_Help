Message 682

Datafile refused at destination.

Problem:

The data file cannot be opened. This can occur for several reasons:

  • The disk is full.
  • The path specified is not correct.
  • You may not have write privileges.
  • There is a communication problem between the instrument and the controller.

Solution:

Try one or more of the following solutions:

  • Delete files from the disk to make room for more files.
  • Reselect the filename using the Experimental View browse button, .
  • Obtain write privileges to the file.
  • Check the cable connections, check the IP address.
  • If communication is re-established during the run no action is required. However, if the problem persists, DO NOT continue any other runs before transferring the current data or the run data will be lost.
  • After communication is established. Use the Tools/Data Transfer function to transfer the data from the instrument to the controller.
  • If you cannot establish communication again, contact TA Instruments for help.
