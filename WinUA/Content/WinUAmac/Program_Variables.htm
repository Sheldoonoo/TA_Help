Predefined Program Variables

 

Direct access to a number of program variables is available through the macro system. Program variables are predefined when a macro runs and do not require initialization (see definitions below). A program variable can be used in exactly the same way as a user defined variable except that the value of the variable is read or written directly from the current open data file in the program.

 

The following variables are predefined:

 

x_cursor[i]  = The last value of the ith x-axis cursor

y_cursor[i]  = The last value of the ith y-axis cursor

i_cursor[i]  = The data point index of the ith cursor (read only)

x_data[i]  = The x-axis value of the ith data point

y_data[i]  = The y-axis value of the ith data point

i_start  = The point index of the first point in the analysis range

i_stop  = The point index of the last point in the analysis range

i_max  = The point index of the last data point.

 

For program variables that have an index, the index of the first value is always 1. The maximum index for cursor variables is 6 and the maximum index for data variables is the value of i_max. Variable indexes may be constants or simple variables.

 

To set the position of a cursor in a subsequent analysis step, assign new values to one or more of the x_cursor and/or y_cursor variables prior to executing the analysis step. Be sure to remove any conflicting Macro Cursor steps that come after the analysis step so they do not overwrite the previously assigned cursor position.

 

NOTE: Use caution when assigning values to program variables. Assigning inappropriate values can cause program errors or invalid results.

 

NOTE: Program variables will only appear in the Macro Watch List Window if they are directly referenced by a macro. If you want to monitor the value of a program variable in the watch window, without actually reading or writing the variable, then you can do so by inserting a Test Variable macro step anywhere in your macro that simply jumps, under any condition, to the next step in the macro. The inserted test variable step will not affect the execution of the macro but will establish a reference to the program variable so that it can be added to the macro watch window.
