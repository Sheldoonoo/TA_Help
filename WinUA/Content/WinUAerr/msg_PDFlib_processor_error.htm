Cannot Create PDF File

PDFlib processor error.

Cannot create PDF file.

 

Problem:

This message is displayed if an error occurs during PDF file creation using Universal Analysis. The most common cause of the problem is a missing or unregistered dynamic link library file called pdflib_com.dll or pdflib_com_5.dll.

 

Solution:

For versions up to 3.9, search for the file pdflib_com.dll. For versions 4.0 or greater, search for the file pdflib_com_5.dll.

 

  • If the file does not exist, it can be reinstalled by reinstalling the Universal Analysis software.

  • If the file exists, it may not be registered. Should that be the case, try registering the file by typing "Regsvr32 pdflib_com" or "Regsvr32 pdflib_com_5," as appropriate, at a system command prompt.

 

The file pdflib_com.dll or pdflib_com_5.dll must appear in the normal executable search path for the system or in the same directory as the file WinUA.exe.
