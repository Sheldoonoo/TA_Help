Main Options Window  Initialization Page

Use this window to specify the file path for the initialization (.ini) file(s) that will be used to define program preferences by the Universal Analysis program and to specify localization language settings.

Initialization File

An initialization file is a file containing all of the settings that are automatically used when the program runs. When you click the Save button on the windows located through Universal Analysis, these selections are saved within the .ini file specified on this Main Options page.

  •  To specify (load) a new initialization file, use the Load button.

  •  To save the current settings to a new filename, select the Save As button and specify a directory location and new filename. This file now becomes the current .ini file, which will contain all future saved settings.

  •  To use the program preferences that were set at TA Instruments (factory settings), click the Load Default Program Preferences button. This will load the factory default settings for the entire program.

  •  Check the Allow multiple sessions of Universal Analysis box to open a new session of the program each time the icon is selected. NOTE: You must save this setting to the current initialization file by clicking the Save button or it will not take effect.

  •  Check the Create Integrity item access files box to generate item access script files for use with the Integrity database system. The files are stored in the directory containing the Universal Analysis program executable file (WinUA.exe). Refer to the TA System Manager program for more information on using these files.

  •  Check the Prompt for Integrity Login box to automatically prompt for login to the Integrity database system when the program is started.

  • Check the Use XP Themes box to have Universal Analysis display windows and dialogs using the currently selected XP Theme on a Windows® XP system. Uncheck this box to display windows and dialogs in Windows classic style. This checkbox has no effect on Windows® NT and Windows® 2000 systems.

  • Click the Save button, if you want to use this .ini file as your new setting or click the OK button to use this setting for this session only.

NOTE: The Save button only stores the settings for the selected page in the initialization (.ini) file. To save all settings, access theMain Options  Initialization Pageand click the Save As button. The Default button will also only recall the default factory settings for the selected page.

 

NOTE: The default directory and filename is TA\WinUA.ini for all versions of Universal Analysis up to and including Version 3.9. Starting with Version 4.0, the default directory and filename is TA\WinUA4.ini.

Localization Language

Universal Analysis can support up to two localization languages at one time. The first language is US English (Windows® language code ENU). Text for the English language is built into the program and does not require any additional language files. English is the default language for Universal Analysis.

 

The second language is a user specified "Other" language. The Other language also defaults to English, but can be changed to any language that has a valid language string. For example, the other language could be German, Japanese, Chinese, etc.

 

If you want to use a language for which there is no language string file available, you can create your own language string file by translating the program's English text strings into text strings of your language (See "Creating a Language String File" in the How To? Help accessed through the Universal Analysis Help menu).

 

Follow these steps to set up the language options used in the program:

  1. Select the radio button for the desired language, either English or Other. If you select the Other button, click on the drop-down list and select the desired language from the Other Language list.

  2. If the language you wish to use is not displayed in the list, click theAdd buttonto add a different language. To remove a language from the language list click theRemove button.

  3. If you want to change the font used for the selected language, click the Font button and select a new font.

 

Language for Hints

The language hint feature provides either flyover help or a status bar help message for most of the text displayed in Universal Analysis. This can be very helpful for someone who is not familiar with the currently selected localization language, but still needs to understand text within the program. The hint can be displayed in English or the "Other" language. The hint appears when the mouse pointer is positioned over a dialog text string, a tool bar icon, or a menu item. Menu item hints appear in the status bar at the bottom of the window.

 

Follow these steps:

  1. If you wish to use the language hint system, check the Show Hint box.

  2. Choose the radio button for the desired language that will be used when the flyover help or status bar help is displayed: English hints or Other language hints.

  3. Check the Show String Number box to append the language string number(s) to the beginning of the hint text. This is useful to determine which string in a language file is used to generate a particular text string in the program. The same string numbers apply to the built-in English strings.

 

NOTE: Hints for the program's main toolbar will still appear when theShow Hint boxis unchecked if theFlyover Help boxis checked on theMain Options  Tool Bar Page.

 

NOTE: You can have Universal Analysis display text in English and hints in the "Other" language by selecting theEnglish language button, theOther language hints button, and theShow Hint box.
