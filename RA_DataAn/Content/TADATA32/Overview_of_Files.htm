Session Files

Session files allow you to record all information active in the data analysis software. When you load a previously saved session, the software loads back all results files and applies all your previous actions, effectively putting you back to where you where before.

 

Session files have the extension SSDA.

Database files allow you to store and retrieve information in analysis databases. Analysis database files have the extension TAA.

Report Writer Format

These files contain information relating to custom reports registered in data analysis. These files are generated as part of the program function and can not be directly manipulated by the user. Report writer format files have the extension CRF. See also:Using the Report Writer

Naming Files

To make it easier to find files, you can use long, descriptive file names. The complete path to the file, including the drive letter, server name, folder path, and file name, can contain up to 255 characters. File names cannot include any of the following characters: forward slash (/), backslash (\), greater than sign (>), less than sign (<), asterisk (*), period (.), question mark (?), quotation mark ("), pipe symbol (|), colon (:), or semicolon (;).
