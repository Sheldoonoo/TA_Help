Viewing the Audit Trail

 

The TA Instruments' Integrity system provides secure, computer-generated, time-stamped audit trails to independently record events.  Each record identifies the person, time/date, and a description of the event.  Comments may be added, if desired, by the user.

The Audit Trail can be viewed from either the Advantage Instrument Control program or Universal Analysis program by selecting View/Audit Trail from the menu.  This window can be used to view audit trail events and add comments to an audit trail.  The events shown in this window can be sorted to enhance locating records using the Set Sort Order, , functions.

The Audit Trail Viewer is divided into two panes: the tree view pane and the table view pane. The left-hand pane displays a tree view representation of all the events. See also:Browsing the Audit Trail Using Tree View

The right-hand pane shows a table view for the selected branch in the tree view.  Unless an audit entry is selected, in which case a textural display of this entry is shown.  The information displayed in the table includes the following information:

  • Module name - specific instrument name (e.g., 0500-0001); Universal Analysis (e.g., WINUA)
  • Description - audit trail event description
  • Project - associated project, if applicable.  If there is not an associated project, "not assigned" will be displayed
  • Event type - numerical assignment
  • Event time - time the event occurred
  • Log time - time the event was logged in the audit trail
  • User ID - log in user ID
  • Added by - full user name
  • Record ID - global audit trail events are assigned "1".  Events associated with the creation of raw data, results or reports records will display the associated Record ID.
  • Comments - user entered comments.  See Adding Comments to the Audit Trail

The order in which these columns are displayed can be changed by "dragging" the column header to the desired column position than "dropping" it in place.

The left-hand pane of the window can be used to organize the events in tree view. Click on each line item to open the items below it until you come to the desired group of events. Browsing the Audit Trail by Record ID

To browse around the audit trail log to locate a particular record ID you can use several different methods:

  • Using the Record Search Option:  To display all of the audit events associated with a specific record ID enter the number of the record you wish to locate into the Record No. field at the top of the Audit Trail Viewer and click the Go button to begin the search.
  • Using the Pop-up Menu:  If you wish to search for records that are related, you have several options to choose from as follows:
  1. Open the TA Audit Trail Viewer.
  2. Select the desired record.
  3. Right click to display the pop-up menu.
  4. Choose from the following menu items depending on the course of action desired:
    • Display entries with the same record ID:  Used to locate an audit trail event with the same record ID as the one selected in step 2.
    • Copy record ID to the clipboard:  Used to copy the current record ID to the clipboard so it can be pasted into the Record No. field at the top of the window.
    • Locate data source for record ID:  Used to find the original data files associated with the current audit event. This automatically opens the Database Viewer window.
    • Reset list view headers:  Used to change the viewing order of the column headings to their default setting in the Audit Trail Viewer window.

Recorded Audit Trail Events

Various events are recorded in the Audit Trail depending upon which Integrity-licensed program is in use:   Instrument Control, Universal Analysis, Rheology Data Analysis Events, or TA System Manager.  This section provides a description of the recorded events available through the Audit Trail Viewer.

Back to top

Instrument Control Events

The recorded events for the Instrument Control program include:

  • Logging in/logging out of the programs (including automatic log outs)
  • All instrument messages
  • Calibration changes (including rheology zero gap and rheology temperature system changes)
  • Run start and completion events
  • Rheology run override (temperature, equilibration, etc.) and abort
  • Rheology run results stored
  • Thermal Analysis Q Series Autosampler pan load/unloads
  • Modifications to the running experiment (e.g., method or gas changes and add or modify step)
  • Procedures (file open and save)
  • Q Series method or sequence file open and save
  • Rheology geometry, session open, and save
  • Software version changes
  • Rheology firmware updates.

Details of the experimental conditions (e.g., Thermal analysis method, calibration, and instrument parameters or rheology procedure, geometry, and instrument options) are stored with each saved data record.  This information can be viewed in the Universal Analysis program by selecting View/Parameter Block or viewed in the Rheology Advantage Data Analysis program as you normally view data.

Universal Analysis Events

The recorded events for the Universal Analysis program include:

  • Logging in/logging out of the programs (including automatic logouts)
  • Data record open and exports (e.g., printing, PDF generation)
  • Saved analysis open and save
  • Results and data report open and save
  • PDF open and save
  • Software version changes

Details of the analyses are recorded in the analysis log within the saved analysis, PDF, or BMP image generated from Universal Analysis.  After opening a data record in Universal Analysis, all actions are recorded within the analysis log.  Once your analysis is complete, you can save this record, along with the associated analysis log, back to the database by selecting File/Save Analysis and/or File/Export PDF File.  If the data is not saved, the analysis log is not recorded.  Saved analysis logs can be viewed through the Database Viewer window.

Rheology Data Analysis Events

The recorded events for the Rheology Data Analysis program include:

  • Logging in and out
  • Data record export to disk
  • Session open and saved
  • Report print and commit
  • Software version changes.

TA System Manager Events

The recorded events for the TA System Manager program include:

  • Logging in/logging out of the programs (including automatic logouts)
  • Creation and modification to User Accounts, Projects, System Objects, and Groups
  • Modifications to system settings

 The audit trail for the TA System Manager is only accessed through the TA System Manager by selecting System Settings, then selecting the System Log page. SeeSetting Up the System.
